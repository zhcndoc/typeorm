# TypeORM 的未来

> 作者：[David Hoeck (@dlhck)](https://github.com/dlhck) 和 [Michael Bromley (@michaelbromley)](https://github.com/michaelbromley)

TypeORM 是 Node.js 生态系统中性能卓越、功能丰富且经过严格考验的 ORM 之一，被全球数十万个项目和公司依赖。每周下载量接近 200 万次，它作为关键依赖支撑着无数应用。然而，过去几年里，维护工作显著放缓，导致其忠实社区对项目未来产生了越来越多的不确定性。

我们很高兴地宣布，Michael Bromley 和 David Hoeck 将在我们母公司 [Elevantiq](https://elevantiq.com/) 的支持下，承担起推动 TypeORM 进入下一个阶段的重任。在 Elevantiq，我们专注于企业数字商务解决方案，TypeORM 是我们的关键依赖——不仅用于我们的旗舰开源项目 [Vendure](https://vendure.io/)，也用于我们的许多其他解决方案。鉴于我们对 TypeORM 的依赖以及其活跃社区日益增长的需求，我们看到了回馈社区的机会，确保项目保持活跃、持续维护且安全可靠。

经过与 TypeORM 原始维护者 Umed 和 Dmitry 的讨论，我们已达成协议，将承担项目的维护工作，灵感来源于成功的、以社区为中心的开源项目。其中值得借鉴的典范是 [Tauri 项目](https://github.com/tauri-apps/tauri)，这是一个由 [Daniel](https://github.com/denjell-crabnebula)（我们通过 Michael 和 Vendure 已有合作）共同创立的自治开源项目。我们与 Tauri 的交流为我们设想的组织架构奠定了基础。

## 我们对 TypeORM 未来的愿景

为确保长期的稳定性和治理，我们计划在荷兰通过 [Commons Conservancy](https://commonsconservancy.org/) 建立一个非营利基金会作为 TypeORM 的承载实体。该基金会将由七人组成的董事会领导：Michael 和 David 以及另外五位致力于项目成功的董事会成员。董事会将与包括企业、贡献者和其他依赖 TypeORM 的社区成员组成的**工作组**紧密合作。这种协作机制将帮助指导符合 TypeORM 生态系统需求和目标的战略决策。

## 组织结构

为推动 TypeORM 发展，我们将设立三个核心领域：

- **开发**：由具备各类数据库引擎和适配器经验的核心开发人员领导，专注于 TypeORM 架构以及持续的技术维护和改进。
- **运营**：负责日常需求，包括文档维护和赞助商会计等事项，确保项目顺利运行。
- **社区**：专注于参与、管理和支持日益壮大的社区，通过 Discord 等平台促进合作与交流。

## 扩大赞助和全职开发力量

我们策略的重要部分是增加赞助力度，积极联系公司并与 [OSS Pledge](https://opensourcepledge.com/) 等组织合作。我们的目标是筹集资金并雇佣两名全职开发人员来领导开发团队，确保项目持续进步和维护。

我们相信，凭借此框架，可以为 TypeORM 构建一个可持续的未来。但_这一愿景的成功依赖社区的支持_。我们也将继续与 [Umed](https://github.com/pleerock) 保持紧密联系，确保他的见解和愿景与项目发展保持同步。
